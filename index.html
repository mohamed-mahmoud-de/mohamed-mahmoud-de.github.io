Weather Data Pipeline DEPI Capstone

Building an automated ETL pipeline that extracts daily weather data from a public API, transforms it with Python, and loads it into PostgreSQL using a schema designed for time-series queries and historical comparisons. Orchestrated with Apache Airflow — scheduled DAGs, retry logic, structured logging, and failure alerting built in. Developed as the capstone project for the DEPI Data Engineering track.

02 —
Egypt Jobs Pipeline V2 Shipped

A multi-version ETL pipeline scraping Egyptian tech job listings from Wuzzuf. V1 was a naive Python script. V2 is a full Apache Airflow DAG with 2-hour scheduling, exponential backoff retries, Discord webhook alerting on failure and success, backfill support for historical intervals, and a Streamlit dashboard for browsing and filtering jobs. V3 will rebuild as a Kafka + Spark streaming system. The goal: demonstrate hands-on understanding of why each tool exists by hitting the wall it was built to solve.

Python Apache Airflow PostgreSQL Streamlit BeautifulSoup Docker ETL
↗
03 —
Real-Time Streaming Data Pipeline Building

Currently designing a real-time streaming pipeline using Apache Kafka and Spark Structured Streaming. Planned to ingest event data through Kafka topics, process with Spark Structured Streaming using window-based aggregations, and write to Cassandra for low-latency operational reads and Elasticsearch for search and aggregation. Currently in architecture and producer design phase — the system is being intentionally built as a deep-dive learning project to understand the trade-offs streaming makes versus batch.
